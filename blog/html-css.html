What does it mean to display inline vs inline blocks?

Inline vs. Inline Block elements

The purpose of this post is to explain the difference between Inline Elements and Inline Block elements.

First, lets look at what an inline element is.


Inline Elements

An inline element is one that does not demand its own width of space, rather will sit next to other inline elements, OR will sit within a block level element.

For example, take a look at this syntax:

The highlighted text in this paragraph is an inline element “span” AND appears within the block element which is the div. A reason for using inline elements is that I can use CSS to isolate and style only the inline element in this block (highlighted).


An inline element, as you can see, does not start on a new line, and is embedded within the wrapping element.

Important to note: inline elements DO NOT have width or height properties as it only takes up as much space as necessary. You can however use padding and margins on this inline element, however expect the existing content to be affected as you add these.


Inline Block Elements


One of the main differences between an inline-block and inline element is that display: inline-block allows you to determine the width and height properties on the element, which for example a span above would not let you perform these properties.
Let’s compare this to a block element such as a div, the inline-block does not add a line-break after the element, so the element can sit next to other elements.

Instances where you may choose to do this would be for a navigation bar for example.

You may now be thinking, well what about display: flex?

The only main difference between inline-flex and inline block is that an inline block creates a specific block for each element under its section while maintaining the structure of each element.
